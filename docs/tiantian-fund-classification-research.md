## 一、天天基金分类体系总览

天天基金的分类体系分为**两个层面**：

### 1.1 前端展示分类（帮助中心 / 基金排行页）

帮助中心（help.1234567.com.cn/question_1380.html）公布的官方分类共 **9 个一级分类**：

| 一级分类 | 二级分类 |
|---------|---------|
| **指数型** | 股票指数、固收指数、海外股票指数、其他 |
| **股票型** | （无细分） |
| **混合型** | 偏股、平衡、偏债、灵活、绝对收益 |
| **债券型** | 长债、中短债、混合一级、混合二级、理财 |
| **货币型** | 普通货币、浮动净值 |
| **QDII** | 普通股票、混合偏股、混合平衡、混合灵活、纯债、混合债、REITs、商品、FOF |
| **REITs** | （无细分） |
| **FOF** | 进取型、均衡型、稳健型 |
| **商品型** | （无细分） |

### 1.2 后端API分类（FundGuideapi 实际返回）

天天基金的 FundGuideapi 接口（`https://fund.eastmoney.com/data/FundGuideapi.aspx`）在实际数据中使用的分类体系与前端展示有所不同：

- API 使用 `ft` 参数区分5大类型：`gp`（股票型）、`zq`（债券型）、`hh`（混合型）、`fof`（FOF）、`qdii`（QDII）
- 每只基金的分类通过 `f[3]` 字段返回
- 格式为 `{一级类型}-{二级类型}`，如 `混合型-偏股`、`债券型-利率债`

## 五、FundGuideapi 接口说明

### 5.1 接口地址
```
https://fund.eastmoney.com/data/FundGuideapi.aspx
```

### 5.2 关键参数
| 参数 | 说明 | 可选值 |
|------|------|--------|
| dt | 数据类型 | 0（基金数据） |
| ft | 基金大类 | gp/zq/hh/fof/qdii |
| sc | 排序字段 | 3nzf（近3月涨幅）、1nzf（近1年涨幅）等 |
| st | 排序方向 | desc/asc |
| pi | 页码 | 1,2,3... |
| pn | 每页数量 | 最大5000 |

### 5.3 返回数据格式
返回JSON，`datas` 数组中的每行数据用逗号分隔：
- `f[0]` = 基金代码
- `f[1]` = 基金名称
- `f[2]` = 拼音缩写
- `f[3]` = **分类**（即 t2 值，格式如"债券型-长债"）
- `f[4]` = 今年来收益率
- `f[5]` = 近1周收益率
- `f[6]` = 近1月收益率
- `f[7]` = 近3月收益率
- `f[8]` = 近6月收益率
- `f[9]` = 近1年收益率
- `f[10]` = 近2年收益率
- `f[11]` = 近3年收益率
- `f[12]` = 近5年收益率
- `f[14]` = 属性标志位
- `f[15]` = 净值日期
- `f[16]` = 单位净值

### 6.3 验证结论

现有代码 `CAT_TREE_TT` 的分类值与 FundGuideapi 实际返回的 `f[3]` 值**基本一致**，但存在以下需要注意的点：

1. **货币型基金缺失**：代码的 t0 分类中没有货币型，货币基金被单独处理或未收录
2. **指数型作为独立分类缺失**：帮助中心将"指数型"作为独立一级分类，但API层面指数基金分属不同的一级分类（股票型下的"指数型-股票"、债券型下的"指数型-固收"等）
3. **混合型包含过多子类**：API层面FOF被部分混入了混合型分类（如`FOF-进取型`在混合型API中也返回），代码照搬了API的这一行为
4. **利率债/信用债**：这两个分类在帮助中心文档中未列出，但在API数据中确实存在（已通过基金详情页和API双重验证）

## 十、项目代码修改建议

基于以上研究，对现有代码的建议：

1. **补充货币型基金**：当前 t0 分类缺少货币型，可考虑增加
2. **CAT_TREE_TT 基本正确**：现有分类值与API实际返回的f[3]值一致，无需大幅修改
3. **分类名称映射**：前后端之间（CAT_TREE_TT → 帮助中心展示名称）可以增加一层映射，方便用户理解
4. **利率债/信用债分类确认**：这两个分类在API中确实存在，现有代码已正确收录
5. **分类URL更新**：如代码中有硬编码的 `*_jzzz.html` URL，需更新为 `*_jzzzl.html`
6. **一二级分类分离建议**：API返回的f[3]值（如"指数型-股票"）本身已经包含了分级信息，可以考虑解析第一个"-"前的部分作为一级分类
